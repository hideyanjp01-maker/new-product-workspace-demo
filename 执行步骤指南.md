# 新品工作台 Demo - 一步一步执行指南

## 📋 准备工作

### 步骤 1: 检查 Node.js 环境

打开终端，执行以下命令检查 Node.js 是否已安装：

```bash
node --version
npm --version
```

如果显示版本号，说明已安装。如果没有，请先安装 Node.js (推荐 v18 或更高版本)。

### 步骤 2: 进入项目目录

```bash
cd /Users/hidemac/Documents/cursor/new-product-workspace
```

### 步骤 3: 安装依赖

```bash
npm install
```

这个命令会安装所有需要的依赖包（React、TypeScript、Vite等），可能需要几分钟时间。

## 🚀 启动项目

### 步骤 4: 启动开发服务器

```bash
npm run dev
```

执行后，终端会显示类似以下信息：

```
  VITE v5.0.8  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 步骤 5: 打开浏览器

在浏览器中访问：**http://localhost:5173**

## 🎯 演示流程

### 场景 1: 从新品灵感洞察器开始

1. **访问新品灵感洞察器**
   - 在首页点击"新品灵感洞察器"按钮
   - 或直接访问：http://localhost:5173/new-product-insight

2. **查看市场机会**
   - 页面会显示多个市场机会洞察卡片
   - 每个卡片显示产品名称、市场规模、增长率等信息

3. **推入新品**
   - 点击任意产品卡片的"推入新品"按钮
   - 确认对话框，点击"确认推入"
   - 系统会自动创建新产品并进入企划期

### 场景 2: 企划期工作流

1. **查看产品列表**
   - 点击导航栏的"产品列表"
   - 或访问：http://localhost:5173/products
   - 可以看到所有产品及其状态

2. **市场研究生成目标**
   - 点击状态为"待生成目标"的产品
   - 在顶部角色选择器中选择"市场研究"
   - 点击"生成目标评估"按钮
   - 产品状态变为"待品牌负责人确认"

3. **品牌负责人确认**
   - 在角色选择器中选择"品牌负责人"
   - 点击"确认"按钮（或"驳回"测试驳回流程）
   - 如果确认，产品进入"待电商负责人确认"状态

4. **电商负责人启动**
   - 在角色选择器中选择"电商负责人"
   - 点击"确认启动新品"按钮
   - 产品进入"冷启动期"

### 场景 3: 冷启动期

1. **查看冷启动期产品**
   - 在产品列表中找到状态为"冷启动期"的产品
   - 点击进入详情页

2. **查看指标数据**
   - 页面会显示该角色相关的指标
   - 包括销售额、订单数、转化率等

3. **完成冷启动**
   - 以"电商负责人"角色访问产品
   - 点击"冷启动完成"按钮
   - 产品进入"放量期"

### 场景 4: 放量期

1. **查看完整数据**
   - 产品进入放量期后，所有角色都可以查看
   - 显示完整的指标数据、AI洞察、目标对比等

2. **切换角色视角**
   - 使用顶部的角色选择器切换不同角色
   - 不同角色会看到不同的指标和内容

### 场景 5: 查看角色工作台

1. **访问角色页面**
   - 在首页的"角色工作台"区域
   - 点击任意角色卡片
   - 或访问：http://localhost:5173/roles/[角色ID]

2. **查看角色图片**
   - 页面会显示该角色的设计图片作为背景
   - 图片100%显示，可以查看完整的设计

3. **查看相关指标**
   - 页面会显示该角色相关的指标卡片

### 场景 6: 工作流管理

1. **访问工作流页面**
   - 点击导航栏的"工作流"
   - 或访问：http://localhost:5173/workflow

2. **查看工作流步骤**
   - 页面显示18个完整的工作流步骤
   - 每个步骤显示状态和描述

3. **执行工作流**
   - 填写当前步骤的表单
   - 点击"完成此步骤"或"标记失败"
   - 系统会自动进入下一步

## 🛠️ 常见问题

### Q1: npm install 失败
**解决方案：**
- 检查网络连接
- 尝试使用国内镜像：`npm install --registry=https://registry.npmmirror.com`
- 清除缓存：`npm cache clean --force`

### Q2: 端口被占用
**解决方案：**
- 修改 `vite.config.ts` 中的端口号
- 或关闭占用5173端口的程序

### Q3: 图片不显示
**解决方案：**
- 检查 `public/images/` 目录下是否有图片文件
- 检查图片路径是否正确
- 清除浏览器缓存

### Q4: 页面空白或报错
**解决方案：**
- 打开浏览器开发者工具（F12）查看控制台错误
- 检查终端是否有编译错误
- 尝试重启开发服务器
